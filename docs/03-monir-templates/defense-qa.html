سوال و جواب دفاع - Templates

تمرین برای توضیح قالب‌ها از دید frontend، backend، database و خروجی رسمی سند.

Concept و Architecture

۱. Template در این پروژه چیست؟

جواب کوتاه: نقشه ظاهری سند رسمی است. جواب عمیق: template مشخص می‌کند محتوا، لوگو، جدول، امضا، QR و شماره رسمی در صفحه چگونه قرار بگیرند.

۲. چرا محتوا و ظاهر را جدا کردید؟

چون کارمند باید متن سند را وارد کند، اما شکل رسمی سند باید توسط قالب قابل مدیریت باشد. این جداسازی تغییر طراحی را بدون تغییر منطق سند ممکن می‌سازد.

۳. lifecycle قالب چیست؟

ساخت قالب، ویرایش layout، ذخیره نسخه، submit، review، approve، binding، preview و render به HTML/PDF.

۴. چرا template version دارید؟

تا تغییرات قالب تاریخچه داشته باشد و نسخه approved از نسخه draft جدا بماند. این برای اسناد رسمی مهم است.

۵. اگر استاد بپرسد بخش شما کدام فایل‌هاست؟

صفحه‌ها در client/src/pages/admin/templates، API در client/src/api/templates.ts، backend در server/src/modules/templates، schema در migration دوم.

Frontend Builder

۶. Template Builder از چه بخش‌هایی ساخته شده؟

Canvas برای صفحه، ribbon برای ابزارها، rail/inspector برای تنظیمات، preview برای دیدن نتیجه و model helperها برای ساخت layout سالم.

۷. TemplateLayout چه مفهوم دارد؟

یک object ساختاری است که اندازه صفحه، جهت متن، blockها، zones و تنظیمات سند را نگه می‌دارد. renderer همین object را می‌خواند.

۸. Block چیست؟

یک عنصر داخل قالب است، مثل متن، تصویر، جدول، QR، خط یا محل امضا. هر block موقعیت، اندازه و style خود را دارد.

۹. Token چیست؟

جای‌نگهدار برای داده متحرک است، مثلا تاریخ سند، شماره مسلسل، نام واحد یا امضای شخص. هنگام render مقدار واقعی جایگزین می‌شود.

۱۰. چرا preview لازم است؟

چون ادمین باید قبل از approve یا چاپ ببیند سند چگونه دیده می‌شود. Preview خطاهای layout را زودتر آشکار می‌کند.

۱۱. چرا A4 مهم است؟

اسناد رسمی معمولا چاپ می‌شوند. پس طراحی در browser باید تا حد ممکن با صفحه چاپی A4 هماهنگ باشد.

۱۲. تصویر لوگو چگونه وارد قالب می‌شود؟

از طریق asset upload در template API. فایل ذخیره می‌شود و layout فقط reference یا metadata آن را نگه می‌دارد.

Backend و API

۱۳. templateApi چه کار می‌کند؟

Frontend را به routeهای template وصل می‌کند: list، create، update، preview، submit، clone، render و PDF.

VS Codeclient/src/api/templates.ts:77
۱۴. routeهای template کجا هستند؟

در server/src/modules/templates/template.routes.ts. هم routeهای کاربر و هم routeهای admin template در همین module تعریف شده‌اند.

۱۵. preview route چه کار می‌کند؟

layout موقت را می‌گیرد و بدون اینکه حتما قالب نهایی شود، خروجی preview تولید می‌کند. برای تست سریع builder استفاده می‌شود.

۱۶. submit route چرا جداست؟

چون ذخیره draft با درخواست review فرق دارد. submit یعنی کاربر آماده است قالب بررسی شود.

۱۷. approve و reject چرا admin route هستند؟

چون تایید قالب رسمی باید صلاحیت مدیریتی داشته باشد. هر سازنده قالب نباید خودش قالب را رسمی کند.

۱۸. binding چیست؟

وصل کردن یک template approved به document type. بعد از binding، سیستم می‌داند هنگام ساخت یک نوع سند کدام قالب را پیشنهاد کند.

Rendering و PDF

۱۹. Renderer چه ورودی‌هایی دارد؟

layout قالب و context سند، شامل subject، تاریخ، متن، fieldها، امضاها، شماره رسمی و verification URL.

۲۰. HTML render با PDF render چه فرق دارد؟

HTML برای preview و نمایش سریع است؛ PDF برای دانلود، چاپ و آرشیف رسمی مناسب‌تر است.

VS Codeserver/src/modules/templates/template-renderer.ts:1337
۲۱. چرا renderer باید امن باشد؟

چون متن سند و fieldها می‌توانند user input باشند. renderer باید خروجی را کنترل کند تا HTML خراب یا ناامن تولید نشود.

۲۲. QR و verification در template کجا استفاده می‌شوند؟

قالب جای QR را مشخص می‌کند، اما token verification از بخش verification می‌آید. renderer این URL را داخل خروجی رسمی قرار می‌دهد.

Database

۲۳. جدول document_templates برای چیست؟

رکورد اصلی قالب را نگه می‌دارد: owner، status، visibility و current version.

VS Codeserver/migrations/sql/002_document_system.up.sql:261
۲۴. جدول version چه چیزی ذخیره می‌کند؟

layout JSON، شماره version، وضعیت review، submit/review info و تاریخچه قالب.

۲۵. جدول bindings چرا جداست؟

چون یک قالب می‌تواند جدا از نوع سند ساخته شود و بعدا به یک یا چند context وصل گردد. جدا بودن binding انعطاف می‌دهد.

۲۶. layout draft چه فایده دارد؟

برای سندهای در حال کار، تغییرات layout موقت ذخیره می‌شود تا کاربر preview و ادامه کار را از دست ندهد.

Demo و دفاع عملی

۲۷. demo سریع برای Templates چیست؟

یک قالب را باز کنید، یک block یا token اضافه کنید، preview بگیرید و توضیح دهید که layout به backend می‌رود و renderer خروجی تولید می‌کند.

۲۸. اگر preview کار نکند، از کجا شروع می‌کنید؟

اول browser console، بعد payload layout، بعد templateApi.preview، بعد route preview و renderer backend.

۲۹. اگر PDF با preview فرق کند، دلیل چیست؟

PDF engine ممکن است margin، font یا اندازه صفحه را کمی متفاوت interpret کند. باید CSS چاپ و A4 constraints بررسی شود.

۳۰. جمع‌بندی ۳۰ ثانیه‌ای شما چیست؟

«Templates ظاهر سند رسمی را از محتوای سند جدا می‌کند. Builder قالب را به شکل layout JSON می‌سازد، نسخه‌ها قابل review هستند، قالب approved به document type bind می‌شود و renderer از layout و داده سند HTML/PDF رسمی تولید می‌کند.»
