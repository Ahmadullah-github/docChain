این سوال‌ها برای دفاع بخش امضا، شماره رسمی، تایید سند و صدور حضوری آماده شده‌اند.

Signature Concepts

۱. Signature Profile چیست؟

جواب کوتاه: پروفایل امضای کاربر است. جواب عمیق: PIN hash، status، failed attempts و active signature asset را نگه می‌دارد.

VS Codeclient/src/pages/app/SignatureProfilePage.tsx:11
۲. فرق signature asset و signature event چیست؟

asset تصویر ذخیره‌شده امضا است؛ event ثبت می‌کند که این امضا در چه سندی، توسط چه user و assignment استفاده شد.

۳. چرا PIN لازم است؟

برای اینکه هر کسی با حساب باز نتواند امضا بزند. PIN یک تایید دوم برای action حساس امضا است.

۴. چرا PIN را plain ذخیره نمی‌کنیم؟

چون PIN محرمانه است. باید hash ذخیره شود تا اگر دیتابیس لو رفت، PIN اصلی قابل خواندن نباشد.

۵. upload session موبایل برای چیست؟

برای وقتی که کاربر می‌خواهد امضا را با موبایل عکس بگیرد و با token موقت به profile خود وصل کند.

۶. چرا امضا event log دارد؟

برای accountability. بعدا می‌توان دید چه کسی، در چه زمان، با کدام assignment و روی کدام سند امضا کرده است.

Signing Flow و Security

۷. جریان امضای سند چیست؟

signing session ساخته می‌شود، PIN verify می‌شود، signature event ثبت می‌شود، و renderer امضا را در خروجی رسمی نشان می‌دهد.

VS Codeserver/src/modules/signatures/signature.routes.ts:916
۸. task signing با document signing چه فرق دارد؟

document signing مستقیم روی سند است؛ task signing وقتی است که امضا بخشی از یک task workflow باشد و task هم update شود.

۹. failed PIN attempts چرا ذخیره می‌شود؟

برای lockout و تشخیص تلاش‌های مشکوک. اگر چندبار PIN غلط وارد شود، profile موقتا قفل می‌شود.

۱۰. document hash در signature event چه فایده دارد؟

برای پیوند امضا با محتوای همان نسخه سند. اگر محتوا تغییر کند، hash کمک می‌کند تفاوت قابل تشخیص باشد.

۱۱. چرا signature asset encryption مهم است؟

تصویر امضا حساس است. encrypt/store کمک می‌کند فایل خام امضا به سادگی قابل سوءاستفاده نباشد.

Serial Numbers

۱۲. serial rule چیست؟

قانون تولید شماره رسمی سند است؛ format، scope، reset policy و padding را مشخص می‌کند.

۱۳. نمونه format شماره رسمی چیست؟

DOC-{YEAR}-{SEQUENCE} یا ترکیب‌هایی با {ORG} و {DOC}. tokenها هنگام تولید با مقدار واقعی جایگزین می‌شوند.

۱۴. scope در serial یعنی چه؟

مشخص می‌کند sequence برای همه سیستم یکی باشد یا بر اساس سازمان، واحد، نوع سند یا ترکیب آن‌ها جدا شود.

۱۵. reset policy چیست؟

مشخص می‌کند sequence سالانه، ماهانه یا هرگز reset شود.

۱۶. چرا serial assignment جدا ذخیره می‌شود؟

برای اینکه شماره رسمی سند یکتا، قابل audit و قابل پیگیری باشد. سند نباید چند شماره رسمی متضاد بگیرد.

Verification

۱۷. verification token چیست؟

یک token برای تایید اصالت سند است. معمولا در QR یا لینک public استفاده می‌شود.

۱۸. چرا token hash ذخیره می‌شود؟

مانند رمزها، بهتر است مقدار اصلی token در دیتابیس raw ذخیره نشود. سیستم token ورودی را hash و مقایسه می‌کند.

۱۹. public verify route چه کار می‌کند؟

token را می‌گیرد، سند مربوط را پیدا می‌کند و معلومات قابل نمایش برای تایید اصالت را برمی‌گرداند.

VS Codeserver/src/modules/verification/document-verification.routes.ts:39
۲۰. QR code در این بخش چه نقش دارد؟

QR فقط شکل تصویری لینک تایید است. کاربر آن را scan می‌کند و به صفحه verification می‌رود.

Walk-in Issuance

۲۱. Walk-in Issuance چیست؟

جریان صدور سند برای مراجعه‌کننده حضوری که ممکن است user سیستم نباشد.

VS Codeclient/src/pages/app/WalkInIssuancePage.tsx:226
۲۲. چرا external persons داریم؟

چون مراجعه‌کننده حضوری الزاما کارمند یا user سیستم نیست. معلومات او جدا از users ذخیره می‌شود.

۲۳. document issuance request چه نگه می‌دارد؟

نوع سند، requester، subject، taker، handled by assignment/unit، status و ارتباط با document ساخته‌شده.

۲۴. create linked document یعنی چه؟

از معلومات درخواست حضوری، یک document واقعی در سیستم ساخته می‌شود تا از template، serial، print و archive استفاده کند.

۲۵. چرا print event ثبت می‌شود؟

برای اینکه معلوم شود چه وقت، توسط چه assignment و چند نسخه سند چاپ شده است.

۲۶. handover record چیست؟

ثبت اینکه سند چاپ‌شده به چه کسی، چه وقت و توسط کدام کارمند تحویل داده شد.
