صفحه اصلی نقشه کد سوال و جواب

Habibullah Qurbani - Signatures, Verification, Walk-in Issuance

مینی کورس دفاع: امضا، شماره رسمی، تایید سند و صدور حضوری

حبیب‌الله مسؤول توسعه و دفاع این بخش است. بخش مربوط به این عضو تیم رسمی بودن سند را توضیح می‌دهد: profile امضا، PIN، upload session، signing event، serial number، verification QR، درخواست حضوری، چاپ، تحویل و آرشیف.

جمله شروع دفاع: «این بخش ثابت می‌کند سند فقط متن نیست؛ سند رسمی باید امضا، شماره مسلسل، لینک تایید، ثبت چاپ و ثبت تحویل داشته باشد.»

۱. Signature Profile

هر کاربر صاحب امضا باید profile داشته باشد: تصویر امضا، PIN hash، وضعیت profile و asset فعال. هنگام امضا، سیستم PIN را verify و signature event را ثبت می‌کند.

۲. Serial و Verification

شماره رسمی سند از serial rule ساخته می‌شود و verification token/QR کمک می‌کند نسخه چاپی یا PDF قابل تایید باشد.

۳. Walk-in Issuance

برای مراجعه‌کننده حضوری، سیستم معلومات شخص، درخواست سند، سند مرتبط، چاپ، تحویل و آرشیف را مرحله‌به‌مرحله ثبت می‌کند.

لکچر ۱: جریان امضا

Signature image -> PIN -> Signing session -> PIN verification -> Signature event -> Rendered official document

کاربر اول profile امضا می‌سازد. تصویر امضا به عنوان asset ذخیره می‌شود و PIN به شکل hash نگهداری می‌گردد. هنگام امضا، سیستم session می‌سازد، PIN را بررسی می‌کند، event ثبت می‌کند و renderer امضا را در جای مناسب سند نشان می‌دهد.

روش گفتن به استاد: «ما خود امضا را فقط تصویر ساده نمی‌گیریم؛ امضا با user، assignment، PIN verification، document hash و signature event ثبت می‌شود.»

لکچر ۲: Upload از دسکتاپ و موبایل

کاربر می‌تواند تصویر امضا را مستقیم از کمپیوتر انتخاب کند یا upload session بسازد و از موبایل آپلود کند. موبایل برای حالتی مفید است که امضا در گوشی عکس گرفته می‌شود.

لکچر ۳: امنیت امضا

تصویر امضا یک دارایی حساس است. ذخیره signature asset، PIN hash، failed attempts، lockout و event log کمک می‌کند سوءاستفاده کمتر شود و اقدام‌ها قابل پیگیری باشند.

لکچر ۴: Serial Rules

Serial rule format شماره رسمی را تعریف می‌کند، مثلا DOC-{YEAR}-{SEQUENCE}. scope مشخص می‌کند شماره جهانی است یا بر اساس واحد/نوع سند جدا می‌شود. reset policy مشخص می‌کند sequence سالانه، ماهانه یا هرگز reset شود.

لکچر ۵: Verification و QR

برای سند رسمی، سیستم token تایید می‌سازد. QR یا لینک تایید به صفحه public verification می‌رود تا شخص بیرونی بتواند اصالت سند را بررسی کند.

لکچر ۶: Walk-in Issuance

Intake -> Create request -> Create linked document -> Finalize/print -> Handover -> Archive

در مراجعه حضوری، شخص ممکن است user سیستم نباشد. بنابراین معلومات او در external persons ذخیره می‌شود. بعد درخواست صدور سند ثبت می‌گردد، سند وابسته ساخته می‌شود، چاپ ثبت می‌شود، تحویل به گیرنده ثبت می‌گردد و بعد پرونده archive می‌شود.

لکچر ۷: Print و Handover

چاپ فقط کلیک download نیست؛ سیستم باید بداند سند کی چاپ شد، چند نسخه چاپ شد، به چه کسی تحویل داده شد، شماره رسمی چه بود و آیا گیرنده امضا یا شست ثبت کرد یا نه.

لکچر ۸: Archive در Walk-in

Archive آخرین مرحله است. وقتی سند چاپ و تحویل شد، پرونده باید با دلیل و زمان آرشیف ذخیره شود تا بعدا قابل پیگیری باشد.

دیتابیس این بخش

signature_profiles

PIN hash، status، failed attempts و active signature asset.

signature_events

ثبت اینکه چه کسی، با کدام assignment، کدام سند را امضا کرده است.

serial_rules / serial_assignments

قانون شماره‌دهی و شماره رسمی اختصاص‌داده‌شده به سند.

document_verification_tokens

tokenهای public برای تایید اصالت سند.

document_issuance_requests

درخواست‌های صدور حضوری و ارتباط با سند.

document_print_events / document_handover_records

ثبت چاپ و تحویل سند رسمی به مراجعه‌کننده.

چک‌لیست مطالعه حبیب‌الله

بتواند فرق signature profile، signature asset و signature event را بگوید.
بتواند PIN verification و lockout را توضیح دهد.
بتواند serial rule format، scope و reset policy را شرح دهد.
بتواند verification token و QR را با مثال دفاع کند.
بتواند walk-in issuance را از intake تا archive رسم کند.
بتواند جداول print و handover را به accountability وصل کند.

تمرین دفاع شفاهی

«من بخش امضا، تایید و صدور حضوری را توضیح می‌دهم. کاربر signature profile و PIN دارد. هنگام امضا، سیستم PIN را verify می‌کند و signature event ثبت می‌شود. برای سند رسمی serial number و verification token ساخته می‌شود. در walk-in issuance معلومات مراجعه‌کننده، درخواست، سند، چاپ، تحویل و archive ثبت می‌گردد.»
