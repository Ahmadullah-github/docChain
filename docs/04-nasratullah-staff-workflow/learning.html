صفحه اصلی نقشه کد سوال و جواب

Nasratullah Joya - Staff Document Workflow

مینی کورس دفاع: جریان کاری سند برای کارمندان

نصرت‌الله مسؤول توسعه و دفاع این بخش است. بخش مربوط به این عضو تیم جریان واقعی کار اداری را پوشش می‌دهد: inbox، ساخت سند، ویرایش، ارسال، task، ضمیمه، comment، notification، finalize و archive.

جمله شروع دفاع: «من بخش workflow کارمندان را توضیح می‌دهم؛ این بخش نشان می‌دهد سند از draft تا ارسال، بررسی، نهایی‌سازی و آرشیف چگونه حرکت می‌کند.»

۱. Staff Workspace چیست؟

محیط کاری کارمند است. کارمند بر اساس assignment فعال خود work itemها، taskها، سندهای مربوط، notificationها و مسیرهای ارسال را می‌بیند.

۲. Workflow چیست؟

workflow یعنی سند فقط ذخیره نمی‌شود؛ از یک واحد یا شخص به دیگری حرکت می‌کند، task ایجاد می‌شود، پاسخ ثبت می‌گردد و وضعیت سند تغییر می‌کند.

۳. چرا این بخش مهم است؟

چون کاربران عادی پروژه بیشتر همین بخش را استفاده می‌کنند. اگر workflow درست باشد، سیستم واقعیت کار اداری پوهنتون را مدل می‌کند.

لکچر ۱: چرخه اصلی سند

Draft -> Preview -> Send -> Task -> Review/Edit/Sign -> Finalize -> Archive

سند اول draft است. کارمند content و template fields را پر می‌کند. بعد سند می‌تواند به واحد یا شخص دیگر ارسال شود. ارسال معمولا task یا transmission می‌سازد. وقتی کار تکمیل شد، سند finalize و در نهایت archive می‌شود.

روش گفتن به استاد: «status سند فقط متن نمایشی نیست؛ هر status نتیجه یک action در workflow است و روی دکمه‌های قابل انجام اثر می‌گذارد.»

لکچر ۲: Work Inbox

Work Inbox خلاصه کارهای فعال را نشان می‌دهد: taskهای باز، سندهای منتظر اقدام، work itemهای مربوط به واحد و لینک به جزئیات سند. این صفحه برای شروع روز کاری کارمند است.

لکچر ۳: Document Registry

Documents page لیست سندها را با scope، جستجو، filter، sort، saved search و actionهای مثل PDF یا delete مدیریت می‌کند. این صفحه برای پیدا کردن و مرور اسناد است.

لکچر ۴: Create و Edit سند

در صفحه create، کارمند نوع سند و template را انتخاب می‌کند، fieldهای لازم را پر می‌کند، preview می‌گیرد و فایل ضمیمه می‌کند. صفحه edit برای تغییر سند موجود قبل از نهایی شدن استفاده می‌شود.

لکچر ۵: Document Detail

مهم‌ترین صفحه staff است. ارسال سند، complete task، comments، attachments، preview رسمی، PDF، finalize، archive و بعضی جریان‌های امضا در همین صفحه دیده می‌شوند.

لکچر ۶: Tasks

Task یعنی درخواست کار روی سند. هر task می‌تواند actionهای مختلف داشته باشد: review، edit، sign، forward، finalize یا archive. وقتی task تکمیل می‌شود، سیستم status و workflow summary را update می‌کند.

لکچر ۷: Transmissions

Transmission جریان رسمی ارسال سند است. می‌تواند به واحد داخلی، assignment مشخص یا گیرنده بیرونی برود. recipientها status جدا دارند تا دریافت و تکمیل قابل پیگیری باشد.

لکچر ۸: Attachments و Comments

Attachment فایل‌های ضمیمه سند را نگه می‌دارد. Comment مکالمه داخلی درباره سند است. هر دو به سند وصل‌اند و در دفاع باید بگویید برای collaboration اداری استفاده می‌شوند.

لکچر ۹: Notifications

وقتی task یا سند به کارمند مربوط شود، notification ساخته می‌شود. هدف این است که کارمند مجبور نباشد همه سندها را دستی بگردد.

دیتابیس Workflow

documents

رکورد اصلی سند، status، owner، holder، subject، content و official serial.

document_tasks

کارهایی که برای سند به واحد، position یا assignment سپرده می‌شود.

document_workflow_events

تاریخچه actionها و تغییرات workflow.

transmissions

ارسال رسمی سند و recipientهای مربوط.

document_attachments

فایل‌های ضمیمه سند.

notifications

پیام‌های in-app برای کاربر و assignment.

چک‌لیست مطالعه نصرت‌الله

بتواند چرخه draft تا archive را توضیح دهد.
بتواند فرق document، task و transmission را بگوید.
بتواند نقش Document Detail page را توضیح دهد.
بتواند بگوید assignment فعال چرا برای staff actions مهم است.
بتواند مسیر frontend API تا backend route را دنبال کند.
بتواند جداول workflow را با مثال عملی وصل کند.

تمرین دفاع شفاهی

«من بخش Staff Workflow را توضیح می‌دهم. کارمند از Work Inbox کارهای خود را می‌بیند، در Documents سندها را جستجو می‌کند، در Create سند draft می‌سازد، و در Detail سند را ارسال، بررسی، ضمیمه، comment، finalize یا archive می‌کند. backend این actionها را در documents، tasks، workflow events، transmissions و notifications ذخیره می‌کند.»
