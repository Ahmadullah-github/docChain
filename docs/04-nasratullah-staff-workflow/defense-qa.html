سوال و جواب دفاع - Staff Workflow

این سوال‌ها برای دفاع بخش workflow کارمندان آماده شده‌اند.

Concept و Screens

۱. Staff Workflow چیست؟

جواب کوتاه: جریان کار روزانه سندها برای کارمندان. جواب عمیق: ساخت، ارسال، بررسی، task، ضمیمه، comment، finalize و archive سند را مدل می‌کند.

۲. فرق Admin و Staff Workflow چیست؟

Admin ساختار پایه و کاربران را تنظیم می‌کند؛ Staff Workflow از همان ساختار برای انجام کارهای واقعی سند استفاده می‌کند.

۳. Work Inbox چه مشکل را حل می‌کند؟

کارمند همه کارهای مربوط به assignment فعال خود را در یکجا می‌بیند و لازم نیست تمام سندها را دستی جستجو کند.

VS Codeclient/src/pages/app/WorkPage.tsx:151
۴. Documents page برای چیست؟

برای registry سندها: لیست، جستجو، filter، saved search، sort، PDF و actionهای سند.

۵. Document Detail چرا مهم‌ترین صفحه است؟

چون بیشتر actionهای سند در آنجاست: ارسال، task، comment، attachment، preview، PDF، finalize و archive.

Document Lifecycle

۶. چرخه اصلی سند چیست؟

Draft، preview، send، task/review/edit/sign، finalize و archive. هر مرحله status و actionهای قابل انجام را تغییر می‌دهد.

۷. سند draft یعنی چه؟

سندی که ساخته شده اما هنوز رسمی یا نهایی نیست. معمولا قابل ویرایش و preview است.

۸. send چه کار می‌کند؟

سند را به واحد، position یا assignment دیگر می‌فرستد و workflow event/task یا transmission مرتبط ایجاد می‌کند.

VS Codeserver/src/modules/documents/document.routes.ts:1705
۹. finalize یعنی چه؟

یعنی سند به حالت رسمی‌تر می‌رسد و معمولا برای شماره رسمی، امضا، چاپ یا آرشیف آماده می‌شود.

۱۰. archive چرا جدا از delete است؟

delete حذف می‌کند، اما archive نگهداری رسمی و تاریخی است. سند اداری معمولا باید قابل پیگیری بماند.

Tasks و Transmissions

۱۱. task چیست؟

یک کار مشخص روی سند است که به واحد، position یا assignment سپرده می‌شود. task می‌تواند review، edit، sign یا finalize بخواهد.

۱۲. task complete چه پیامدی دارد؟

status task تغییر می‌کند، response ثبت می‌شود و ممکن است status سند یا workflow summary update شود.

VS Codeserver/src/modules/documents/document.routes.ts:2534
۱۳. transmission چیست؟

ثبت رسمی ارسال سند و recipientهای آن. برای پیگیری ارسال، دریافت و تکمیل استفاده می‌شود.

۱۴. فرق task و transmission چیست؟

task کار قابل انجام است؛ transmission record رسمی ارسال است. یک سند می‌تواند هم task داشته باشد هم transmission.

۱۵. چرا recipient status داریم؟

چون هر گیرنده ممکن است در زمان متفاوت سند را دریافت، acknowledge یا complete کند.

Frontend و API

۱۶. documentApi چه کار می‌کند؟

تمام actionهای سند مثل create، list، detail، update، send، finalize، archive، attachment و task را به backend وصل می‌کند.

VS Codeclient/src/api/documents.ts:67
۱۷. Create page چگونه preview می‌گیرد؟

template fields و layout را به template API می‌فرستد تا HTML preview برگردد و کارمند قبل از ذخیره نتیجه را ببیند.

۱۸. attachment upload کجا انجام می‌شود؟

در Document Detail یا Create، فایل با form-data به route upload سند فرستاده می‌شود و به document_attachments وصل می‌گردد.

۱۹. comments چه فایده دارند؟

برای مکالمه داخلی درباره سند. هر comment به document و author assignment وصل است.

۲۰. notifications چگونه به workflow کمک می‌کنند؟

وقتی task یا سند به کارمند مربوط شود، notification او را آگاه می‌کند و سرعت پیگیری را بالا می‌برد.

Backend و Database

۲۱. routeهای سند کجا هستند؟

در server/src/modules/documents/document.routes.ts. این فایل قلب backend سند است.

VS Codeserver/src/modules/documents/document.routes.ts:36
۲۲. جدول documents چه نگه می‌دارد؟

رکورد اصلی سند: subject، content، status، owner/current holder، document type، serial و زمان‌های finalize/archive.

۲۳. چرا document_versions داریم؟

برای نگهداری تاریخچه تغییرات محتوای سند. اگر سند ویرایش شود، نسخه قبلی قابل پیگیری می‌ماند.

۲۴. workflow events برای چیست؟

ثبت timeline actionهای سند، مثل send، return، review، finalize یا archive. این برای audit و نمایش تاریخچه مهم است.

۲۵. چرا actionها به assignment وصل هستند؟

چون در اداره مهم است بدانیم شخص در کدام موقف اداری اقدام کرده، نه فقط با کدام حساب login وارد شده است.

۲۶. workspace routes چه کار می‌کنند؟

summary، work-items و targets را برای dashboard/inbox کارمند آماده می‌کنند.

Demo و Troubleshooting

۲۷. demo سریع برای این بخش چیست؟

یک سند draft بسازید، preview بگیرید، آن را به یک واحد ارسال کنید، task را در detail نشان دهید و سپس مسیر database را توضیح دهید.

۲۸. اگر سند در inbox دیده نشود، چه بررسی می‌کنید؟

assignment فعال، target unit/position، task status، workspace API response و permissionهای سند.

۲۹. اگر کاربر نتواند سند را edit کند، دلیل چیست؟

ممکن است status سند اجازه ندهد، user holder نباشد، task permission edit نداشته باشد، یا سند finalized/archive شده باشد.

۳۰. جمع‌بندی ۳۰ ثانیه‌ای شما چیست؟

«Staff Workflow جریان واقعی سند را مدیریت می‌کند. کارمند سند می‌سازد، preview می‌گیرد، ارسال می‌کند، taskها انجام می‌شود، ضمیمه و comment اضافه می‌شود، و در آخر سند finalize و archive می‌شود. backend این مسیر را در documents، tasks، workflow events، transmissions و notifications نگه می‌دارد.»
